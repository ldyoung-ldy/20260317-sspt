# Step 2 赛事管理手工验收脚本

## 验收目标

验证 Step 2 第一阶段闭环是否成立：

`管理员创建赛事 -> 后台查看 -> 编辑 -> 发布 -> 前台首页可见 -> 详情页可见 -> 取消发布后前台隐藏 -> 草稿安全删除`

## 前置条件

1. 本地开发环境已启动：`bun run dev`
2. 已具备管理员账号并可访问 `/admin`
3. 数据库可写
4. 浏览器使用无痕窗口或新会话，避免旧缓存干扰

## 建议测试数据

- 赛事名称：`AI 创新挑战赛 2026`
- 赛事描述：`面向企业创新团队的 AI 赛事，覆盖报名、作品提交与评审全流程。`
- 赛道：
  - `企业智能体`
  - `智能工作流`
- 赛题：`流程自动化`
- 奖项：`一等奖 / ¥20,000`
- 评分维度：
  - `创新性 / 10 / 40`
  - `完成度 / 10 / 35`
  - `落地价值 / 10 / 25`

## 验收步骤

### 场景 A：创建赛事

1. 进入 `/admin/events/new`
2. 填写基础信息
3. 填写一组合法时间：
   - 报名开始 < 报名截止
   - 报名截止 <= 提交开始
   - 提交开始 < 提交截止
   - 提交截止 <= 评审开始
   - 评审开始 < 评审截止
   - 评审截止 <= 赛事结束
4. 添加至少 1 个赛道
5. 添加至少 1 个奖项
6. 保留至少 1 个评分维度
7. 点击“创建赛事”

**预期结果**

- 跳转回 `/admin/events`
- 新赛事出现在列表中
- 状态为“草稿”
- slug 自动生成
- “待发布”按钮存在，前台预览不可点

### 场景 B：表单校验

1. 再次进入 `/admin/events/new`
2. 故意把 `submissionStart` 设置到 `registrationEnd` 之前
3. 点击提交

**预期结果**

- 页面不跳转
- 表单展示明确校验错误

继续验证：

4. 添加两个同名评分维度
5. 点击提交

**预期结果**

- 页面不跳转
- 出现“评分维度名称不能重复”或等价报错

继续验证：

6. 添加一个 `select` 类型自定义字段但不填选项
7. 点击提交

**预期结果**

- 页面不跳转
- 出现“下拉字段至少需要一个选项”或等价报错

### 场景 C：后台发布赛事

1. 进入 `/admin/events`
2. 找到刚创建的草稿赛事
3. 点击“发布”

**预期结果**

- 状态切换为“已发布”
- 可看到“预览”按钮
- 阶段 badge 可正常显示

### 场景 D：前台首页展示

1. 打开 `/`
2. 查找刚发布的赛事卡片

**预期结果**

- 首页出现该赛事
- 卡片只展示已发布赛事
- 能看到名称、slug、摘要、阶段、时间范围、配置数量、报名截止时间

### 场景 E：赛事详情页展示

1. 点击首页卡片中的“查看详情”
2. 进入 `/events/[slug]`

**预期结果**

- 标题、slug、描述显示正常
- 时间线 4 个区块显示正常
- 赛道、赛题、奖项、评分维度正常展示
- 报名区块根据当前时间显示正确提示

### 场景 F：取消发布回收前台曝光

1. 回到 `/admin/events`
2. 点击“取消发布”
3. 回到 `/`

**预期结果**

- 赛事从首页消失
- 后台列表状态恢复为“草稿”
- “预览”重新不可用

### 场景 G：编辑赛事并验证前后台同步

1. 进入 `/admin/events/[id]/edit`
2. 确认名称、描述、8 个时间字段、赛道/赛题/奖项/评分维度已正确回填
3. 修改以下任一信息：
   - 名称
   - 描述
   - 时间字段
   - 赛道或奖项内容
4. 点击“保存修改”
5. 回到 `/admin/events`，再进入 `/events/[slug]`

**预期结果**

- 编辑页展示的是当前最新数据
- 保存后跳回 `/admin/events`
- 后台列表中的名称 / 摘要 / slug（若名称变更触发）同步更新
- 前台详情页展示同步后的内容
- 时间字段不会因部署机 / 浏览器时区差异发生整体偏移

### 场景 H：安全删除草稿赛事

1. 保持赛事为“草稿”状态
2. 在 `/admin/events` 或 `/admin/events/[id]/edit` 点击“删除草稿”
3. 确认弹窗

**预期结果**

- 仅草稿赛事可见删除按钮
- 删除成功后返回列表页，赛事从后台列表消失
- 若赛事已发布，则不应展示删除按钮
- 若赛事存在报名 / 作品 / 评分 / 评委分配数据，删除应被阻止并提示原因

## 边界回归建议

1. 使用中文名称、英文名称、带符号名称分别创建赛事，检查 slug
2. 创建无赛道 / 无赛题 / 无奖项赛事，确认前台页面不崩
3. 刷新首页与详情页，确认发布切换后内容刷新正常
4. 修改赛事名称后，确认 slug 是否符合预期且旧详情链接行为可接受
